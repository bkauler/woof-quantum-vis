Einige Hinweise zur Terminologie

Die Wörter "Plattenlaufwerk" oder "Festplatte" beziehen sich auf das gesamte physische Laufwerk, wohingegen eine "Partition" ein Teil des Laufwerks ist, das ein "Dateisystem" enthält.

Zum Beispiel kann in der Welt von Unix/Linux ein gesamtes Laufwerk als /dev/hda benannt werden, wohingegen eine Partitionen darinnen als /dev/hda1, /dev/hda2, etc. benannt werden.

Das Problem bei dieser Terminologie ist, dass in der Welt von Windows/DOS das Wort "Laufwerk" sich auf eine Partition bezieht, nicht nicht auf das gesamte Gerät. So ist zum Beispiel "Laufwerk C:" wirklich eine Partition. Es ist korrekter, C: als ein "logisches Laufwerk" zuu bezeichnen, was auch ein in der Welt von Windows/DOS benutzter Ausdruck ist.

Es ist auch nötig, zwischen einer Partition und einem Dateisystem zu unterscheiden, was wiederum in der Welt von Windows/DOS verwischt ist. Man kann auf einer Festplatte eine Partition erzeugen, dann darin ein Dateisystem. Beispiele von Dateisystemen sind msdos, vfat, ntfs, ext2, ext3 und reiserfs.

Auch das DOS-Programm "format" verwirrt. Es erzeugt aktuell ein MSDOS-Dateisystem in einer Partition. Das DOS-Programm "fdisk" kann Partitionen erzeugen und löschen. (Linux hat auch ein Programm "fdisk", und es ist in Puppy)

Laufwerke oder Partitionen kopieren

Wenn ich das Wort "Laufwerk" benutze, meine ich damit normalerweise das gesamte Gerät. Ich benutze "Partition", wenn ich eine Partition meine.

Pudd kann ein ganzes Laufwerk oder eine Partition kopieren. Es gibt hier noch einige Restriktionen, die ich in künftigen Versionen beseitigen will:

  • Ein Laufwerk kann nur auf ein anderes identisches Laufwerk kopiert werden. Man kann zum Beispiel eine 128M Compact-Flash-Karte auf eine andere 128M Compact-Flash-Karte kopieren (Ich weiß nicht, ob es irgendwelche Probleme beim Kopieren zwischen verschiedenen Modellen/Herstellern von CF-Karten gibt, oder bei anderen Typen von Flash-Karten).
  • Eine Partition kann auf eine andere Partition kopiert werden, die mindestens dieselbe Größe hat. Einige hier noch auszuarbeitende Details: Bis jetzt muss es derselbe Typ des Dateisystems sein. Das Quelldateisystem wird was auch immer in der Zielpartition ist, überschreiben. So weit kann Pudd zum Füllen der Zielpartition nur von ext2- und ext3-Dateisystemen die Größe ändern.

Pudd erlaubt das Kopieren von Laufwerk/Partition auf eine Datei, dann könnte man diese Datei auf ein Laufwerk/Partition kopieren. Jedoch können die zwei nicht vermischt werden. Das heißt, man kann nicht ein Laufwerk in eine Datei kopieren und dann Datei auf Partition, da die Datei das Image des gesamten Laufwerks enthalten wird, nicht nur einer einzelnen Partition.
