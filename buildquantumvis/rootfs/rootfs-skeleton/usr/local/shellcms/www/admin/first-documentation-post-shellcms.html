First documentation post, shellCMS

November 16, 2017 — MrPerson

This is an installation of shellCMS, for documentation posts. Documentation posts differ from blog posts, as they are filed by category, whereas blog/news posts are filed by date.

This raw post is filed locally at ~/.shellcms/www/admin/first-documentation-post-shellcms.htmraw, and the generated post (with header, footer, layout, etc.) is at ~/.shellcms/www/admin/first-documentation-post-shellcms.html. It is filed under category "admin".

To find out about blog posts, click on "news" link above.

Images may be inserted in posts. All posts have an images folder, in this case ~/.shellcms/www/admin/images, so just specify a relative path, for example images/fido96.png:

When creating or editing a post, do not worry about layout. Header, footer and layout are taken care of when the post is "posted", that is, the final first-documentation-post-shellcms.html file. For example, if you insert a table into this post, no need to set the width, for example:

First cell of table. Not bothering about width, as it will be taken care of after posting.
 Second cell of table.

Read the test post in the blog ("News" link above) for a bit more info. Also, there is a website for shellCMS, with documentation:

http://bkhome.org/shellcms

Have fun, shellCMS is super easy to use!
