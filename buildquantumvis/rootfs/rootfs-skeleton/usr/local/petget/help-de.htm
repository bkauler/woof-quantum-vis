PKGget Package Manager (PPM)

Ein „Paket“ ist die Linux-Terminologie für eine Anwendung. Diese werden in Online-Repositorys gespeichert und Sie können jedes Paket herunterladen und installieren.

PKGget ist ein Nachkomme des Puppy Package Managers, der auch unter dem Akronym „PPM“ bekannt ist. Der Unterschied besteht darin, dass es sich bei PKGget um eine spätere Generation mit mehr Funktionen handelt.

Repository- und Kategorieauswahl

Sie klicken oben auf ein Optionsfeld, um das gewünschte Repository auszuwählen – normalerweise ist das ganz links dasjenige, das für Ihre aktuelle Version von EasyOS am besten geeignet ist.

Mit den vertikal ausgerichteten Optionsfeldern auf der linken Seite können Sie die Anzeige auf eine bestimmte Kategorie eingrenzen (diese Hinweise gelten für die „Classic“-Benutzeroberfläche, die „Ziggy“-Benutzeroberfläche hat ein etwas anderes Layout).

Beachten Sie die Kontrollkästchen mit der Bezeichnung „EXE“ und „DEV“ (und bei einigen Versionen von PKGget auch „DOC“ und „NLS“). Viele Pakete sind wie folgt in kleinere Pakete aufgeteilt:
EXE Dies ist die eigentliche Anwendung, die Sie auf jeden Fall wollen.
DEV Enthält Entwicklungsdateien, die nur für diejenigen von Interesse sind, die Quellcode kompilieren.

Die meisten Pakete verfügen über integrierte Dokumentations- und Übersetzungsdateien, nicht als separate Pakete. Es hängt vom Paket-Repository ab; Leere Linux-Pakete werden beispielsweise in zwei Teile geteilt, das Hauptpaket und ein „-devel“-Paket.

Ein Beispiel ist „libwv-1.2.3.x86_64.xbps“ und „libwv-devel-1.2.3.x86_64.xbps“. Wenn das Kontrollkästchen „EXE“ aktiviert ist, wird nur das erste Paket in PKGget angezeigt. Das Gleiche gilt für „DEV“. Oder aktivieren Sie beide Kontrollkästchen, um beide Pakete anzuzeigen.

So installieren und deinstallieren Sie ein Paket

Wenn Sie auf eines der im großen Fenster angezeigten Pakete klicken, öffnet sich ein weiteres Fenster, in dem Sie gefragt werden, ob Sie das Paket wirklich installieren möchten, und in dem auch weitere Informationen zum Paket angezeigt werden – so können Sie mehr darüber erfahren, bevor Sie sich für die Installation entscheiden Es.

Wenn ein Paket installiert ist, erscheint es im Fenster unten rechts.

Um ein Paket zu deinstallieren, klicken Sie einfach darauf. Es erscheint ein weiteres Fenster, in dem Sie aufgefordert werden, den Löschvorgang zu bestätigen.

Paket-Repositorys

Wie oben erwähnt, ermöglichen Optionsfelder oben die Auswahl eines Repositorys.

Normalerweise handelt es sich dabei um die „kompatible Distribution“, die zum Erstellen von EasyOS verwendet wurde, sowie um ein oder mehrere PET-Repositorys.

Zum Beispiel die Kirkstone-Reihe von EasyOS; Die Pakete wurden in OpenEmbedded kompiliert und die resultierenden Binärpakete zum Erstellen von EasyOS verwendet. Alle in OpenEmbedded kompilierten Pakete werden ins Internet hochgeladen und als Paket-Repository bereitgestellt. Sie sehen einen Radiobutton mit der Bezeichnung „oe-kirkstone“.

Ein weiteres Beispiel ist EasyOS, das mit Void-Linux-Paketen erstellt wurde. In diesem Fall sehen Sie die Optionsfelder „void-current“ und „void-current-nonfree“.

Es gibt verschiedene PET-Repositorien. PETs sind das Puppy-Format für Pakete, ein Akronym, das wir humorvoll für Puppy Extra Treats sagen. Welche angeboten werden, hängt davon ab, welche EasyOS-Serie Sie verwenden, aber normalerweise gibt es „pet-noarch“ und andere wie „pet-kirkstone“ oder „pet-void“.

„pet-noarch“ sind keine Binärpakete. Das heißt, sie enthalten keine ausführbaren Binärdateien, die eine bestimmte CPU-Architektur wie x86_64 oder aarch64 erfordern. Diese Pakete laufen auf jeder CPU.

„pet-kirkstone“ und „pet-void“ sind Binärpakete.

PKGget ist ein Chamäleon, das Pakete aus fast jedem Repository installieren kann. Aus Kompatibilitätsgründen würden Sie jedoch normalerweise nur von denen installieren, die in den Optionsfeldern des Repositorys angezeigt werden.

Zu Ihrer Information: Die Pakete verschiedener Linux-Distributionen können anhand der Dateinamenerweiterung identifiziert werden:
Debian, Ubuntu „.deb“
Slackware „.tgz“ oder „.txz“
Mageia, Fedora „.rpm“
Arch „.pkg.tar.xz“ oder „.pkg.tar.zst“
Ungültig „.xbps“
Haustier Haustier"

PKGget ist in der Lage, all diese zu installieren. Sie können tatsächlich jedes fremde Paket installieren; Laden Sie beispielsweise ein „.deb“-Paket herunter, klicken Sie darauf und wählen Sie die Installation. Aber hier muss man natürlich vorsichtig sein, da ein ausländisches Paket möglicherweise nicht kompatibel ist.

Suchen

Über das Feld „Suchen:“ können Sie nach einem beliebigen Paket suchen.

Sie können einen Teil des Namens eingeben, eine Übereinstimmung wird jedoch nur auf der linken Seite gefunden. Wenn Sie beispielsweise nach „pan“ suchen, finden Sie möglicherweise Folgendes:

pango-1.28.1-w5 Bibliothek für Layout und Darstellung internationalisierter Texte
pangomm-2.26.2-w5 oo Wrapper für die Pango-Bibliothek
panoramixproto-1.1-w5 Panoramix-Erweiterungsheader

Eine Suche nach „mm“ führt jedoch zu keinem Ergebnis, da es keine Paketnamen gibt, die mit „mm“ beginnen.

Das Feld „Suchen“ akzeptiert jedoch „*“-Platzhalter; sogenannte „Glob-Wildcards“. Wenn Sie also „*mm“ eingeben, werden alle Pakete gefunden, deren Namen „mm“ enthalten.

Sie können sogar mehrere Platzhalter verwenden, zum Beispiel „p*mix*to“, wodurch das Paket „panoramixproto“ gefunden wird.

Bei der Suche wird die Groß-/Kleinschreibung nicht beachtet, daher liefert eine Suche nach „PAN“ die gleichen Ergebnisse wie oben gezeigt.

PKGget unterstützt auch die Suche nach mehreren Schlüsselwörtern. Außerdem wird das Feld Beschreibung sowie der Paketname durchsucht. Wenn Sie beispielsweise „Abiword-Plugins“ in das Suchfeld eingeben, werden Pakete mit diesen beiden Wörtern gefunden, entweder im Paketnamen oder im Description.

Bei der Suche ist zu beachten, dass sie das gesamte Repository (oder die gesamten Repositorys) durchsucht und die vertikalen Kategorie-Radiobuttons auf der linken Seite ignoriert.

Installation ohne die PKGget-GUI

Dies wurde bereits oben erwähnt; Das Beispiel zum Herunterladen eines .deb-Pakets.

Sie können tatsächlich jedes Paket von überall herunterladen und installieren. Alles, was Sie tun müssen, ist auf das Paket im Dateimanagerfenster zu klicken, und die Paketinstallationskomponente von PKGget wird das Paket ausführen und installieren. Dies funktioniert für alle Arten von Paketen, einschließlich PETs, Ubuntu/Debian DEBs, Slackware, Arch und Void-Pakete.

Beachten Sie jedoch, dass bei der Installation eines einzelnen Pakets auf diese Weise, ohne die PKGget-GUI-Schnittstelle zu verwenden, keine Abhängigkeiten installiert werden und Sie sich selbst darum kümmern müssen.

Das Gute daran ist, dass ein auf diese Weise installiertes Paket im Fenster „Installierte Pakete“ der PKGget-GUI angezeigt wird und somit deinstalliert werden kann.

So aktualisieren Sie EasyOS auf die nächste Version

Vor einiger Zeit kritisierte jemand PPM und meinte, dass es kein vollständiges Upgrade von Puppy (oder EasyOS) auf die nächste Version ermögliche. Mit den Paketmanagern einiger Distributionen können Sie alle Pakete installieren, die für ein vollständiges Update auf die neueste Version der Distribution erforderlich sind.

PPM bietet das nicht, da Puppy (und EasyOS) nicht so funktionieren. Es liegt außerhalb des Geltungsbereichs, oder wir könnten sagen, des „Vertrags“ dessen, wozu PPM verpflichtet ist.

In EasyOS besteht die übliche Möglichkeit, ein Versionsupdate durchzuführen, darin, auf dem Desktop auf das Symbol „Update“ zu klicken. Lesen Sie hier mehr: https://easyos.org/user/how-to-update-easyos.html
