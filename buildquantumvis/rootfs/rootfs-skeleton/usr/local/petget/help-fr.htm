PKGget Package Manager (PPM)

Un « package » est la terminologie Linux désignant une application. Ceux-ci sont conservés dans des référentiels en ligne et vous pouvez télécharger et installer n'importe quel package.

PKGget est un descendant de Puppy Package Manager, également connu sous l'acronyme « PPM ». La différence est que PKGget est une génération ultérieure, avec plus de fonctionnalités.

Sélection du référentiel et des catégories

Vous cliquez sur un bouton radio en haut pour choisir le référentiel souhaité - généralement celui le plus à gauche est celui le plus adapté à votre version actuelle d'EasyOS.

Les boutons radio alignés verticalement sur la gauche vous permettent de restreindre l'affichage à une catégorie spécifique (ces notes concernent l'interface utilisateur 'Classic', l'interface utilisateur 'Ziggy' a une présentation légèrement différente).

Notez les cases à cocher « EXE » et « DEV » (et sur certaines versions de PKGget également « DOC » et « NLS »). De nombreux packages sont divisés en packages plus petits, comme suit :
EXE C'est l'application réelle, vous la voulez absolument.
DEV Il contient des fichiers de développement, qui n'intéressent que ceux qui compilent le code source.

La plupart des packages contiennent de la documentation et des fichiers de traduction intégrés, et non des packages séparés. Cela dépend du référentiel de packages ; Les packages Void Linux, par exemple, sont divisés en deux, le package principal et un package "-devel".

Un exemple est « libwv-1.2.3.x86_64.xbps » et « libwv-devel-1.2.3.x86_64.xbps ». Si la case "EXE" est cochée, alors seul le premier package s'affichera dans PKGget. Idem pour "DEV". Ou cochez les deux cases pour afficher les deux packages.

Comment installer et désinstaller un package

Lorsque vous cliquez sur l'un des packages affichés dans la grande fenêtre, une autre fenêtre apparaît vous demandant si vous souhaitez vraiment installer le package, et propose également des informations supplémentaires sur le package - afin que vous puissiez en savoir plus avant de décider de l'installer. il.

Lorsqu'un package est installé, il apparaît dans la fenêtre en bas à droite.

Pour désinstaller un package, cliquez simplement dessus. Il y aura une autre fenêtre vous demandant de confirmer la suppression.

Dépôts de packages

Comme mentionné ci-dessus, les boutons radio en haut permettent la sélection d'un référentiel.

Habituellement, il s'agira de la "distribution compatible" utilisée pour créer EasyOS, ainsi que d'un ou plusieurs référentiels PET.

Par exemple, la série Kirkstone d'EasyOS ; les packages ont été compilés dans OpenEmbedded et les packages binaires résultants ont été utilisés pour créer EasyOS. Tous les packages compilés dans OpenEmbedded sont téléchargés sur Internet et fournis sous forme de référentiel de packages. Vous verrez un bouton radio intitulé "oe-kirkstone".

Un autre exemple est EasyOS construit avec les packages Void Linux. Dans ce cas, vous verrez les boutons radio "void-current" et "void-current-nonfree".

Il existe différents référentiels PET. Les PET sont le format Puppy pour les emballages, un acronyme qui, selon nous, signifie avec humour Puppy Extra Treats. Ceux proposés dépendront de la série EasyOS que vous utilisez, mais il y aura généralement "pet-noarch" et un autre tel que "pet-kirkstone" ou "pet-void".

"pet-noarch" ne sont pas des packages binaires. Autrement dit, ils ne contiennent pas d'exécutables binaires nécessitant une architecture CPU spécifique telle que x86_64 ou aarch64. Ces packages fonctionneront sur n’importe quel processeur.

"pet-kirkstone" et "pet-void" sont des packages binaires.

PKGget est un caméléon, capable d'installer des packages depuis presque n'importe quel référentiel. Cependant, pour des raisons de compatibilité, vous n'installerez normalement qu'à partir de ceux affichés dans les boutons radio du référentiel.

Pour information, les packages des différentes distributions Linux peuvent être identifiés par l'extension de nom de fichier :
Debian, Ubuntu ".deb"
Slackware ".tgz" ou ".txz"
Mageia, Fedora ".rpm"
Arch ".pkg.tar.xz" ou ".pkg.tar.zst"
Annuler ".xbps"
ANIMAUX DE COMPAGNIE ".pet"

PKGget est capable d'installer n'importe lequel d'entre eux. Vous pouvez réellement installer n’importe quel package étranger ; par exemple, téléchargez un package ".deb", cliquez dessus et choisissez d'installer. Mais bien sûr, vous devez faire attention à cela, car un package étranger peut ne pas être compatible.

Recherche

La case « Rechercher : » vous permet de rechercher n'importe quel package.

Vous pouvez saisir un nom partiel, mais une correspondance n'est trouvée qu'à partir de la gauche. Par exemple, si vous effectuez une recherche sur « pan », vous pourriez trouver ceci :

pango-1.28.1-w5 Bibliothèque pour la mise en page et le rendu du texte internationalisé
pangomm-2.26.2-w5 oo wrapper pour la bibliothèque pango
panoramixproto-1.1-w5 En-têtes d'extension Panoramix

Mais faire une recherche sur "mm" ne donne rien, car il n'y a aucun nom de package commençant par "mm".

Cependant, la zone Rechercher acceptera les caractères génériques « * » ; ce que l'on appelle des "caractères génériques globaux". Alors, tapez "*mm" pour trouver tous les packages avec "mm" dans leur nom.

Vous pouvez même avoir plusieurs caractères génériques, par exemple "p*mix*to", qui trouveront le package 'panoramixproto'.

La recherche n'est pas sensible à la casse, donc une recherche sur "PAN" donnera les mêmes résultats que ceux indiqués ci-dessus.

PKGget prend également en charge la recherche de plusieurs mots-clés. En outre, le champ Description est recherché ainsi que le nom du package. Par exemple, si vous tapez dans le champ de recherche "plugins abiword", il trouvera des packages contenant ces deux mots, soit dans le nom du package, soit dans la description.

Une chose à noter à propos de la recherche est qu'elle recherche dans l'ensemble du référentiel (ou des référentiels) et ignore les boutons radio verticaux Catégorie sur le côté gauche.

Installation sans l'interface graphique de PKGget

Déjà mentionné ci-dessus ; l'exemple donné de téléchargement d'un package .deb.

Vous pouvez réellement télécharger n’importe quel package, de n’importe où, et l’installer. Tout ce que vous avez à faire est de cliquer sur le package dans la fenêtre du gestionnaire de fichiers, et le composant d'installation du package de PKGget exécutera et installera le package. Cela fonctionne pour tous les types de packages, y compris les packages PET, Ubuntu/Debian DEB, Slackware, Arch et Void.

Notez cependant que si vous installez un package individuel de cette manière, sans passer par l'interface graphique de PKGget, aucune dépendance n'est installée et vous devrez vous en occuper vous-même.

Une bonne chose, un package installé de cette manière apparaîtra dans la fenêtre "Packages installés" de l'interface graphique PKGget et il pourra ainsi être désinstallé.

Comment mettre à jour EasyOS vers la version suivante

Il y a quelque temps, quelqu'un a critiqué PPM, affirmant qu'il ne permettait pas une mise à niveau complète de Puppy (ou EasyOS) vers la version suivante. Les gestionnaires de packages de certaines distributions vous permettent d'installer tous les packages requis pour une mise à jour complète vers la dernière version de la distribution.

PPM ne propose pas cela, car Puppy (et EasyOS) ne fonctionnent pas de cette façon. Cela sort du cadre, ou pourrions-nous dire du « contrat » de ce que PPM est tenu de faire.

Dans EasyOS, la manière habituelle d'effectuer une mise à jour de version est de cliquer sur l'icône « mettre à jour » sur le bureau. En savoir plus ici : https://easyos.org/user/how-to-update-easyos.html
